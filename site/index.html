The Effectiveness of Supervised Machine Learning Algorithms in Predicting Software Refactoring

Information

Refactoring is the process of changing a software system in a way that does not alter the external behaviour of the code, yet improves its internal structure.

We explore the effectiveness of machine learning models to predict software refactorings. More specifically, we make use of six different machine learning algorithms (i.e., Logistic Regression, Naive Bayes, SVM, Decision Trees, Random Forest, and Deep Neural Network) and train them in a dataset of more than two million real-world refactorings that happened in 11,149 projects from the Apache, F-Droid, and GitHub ecosystems.

Accuracy:

Our models are able to predict 20 different refactoring operations at class-, method-, and variable-levels, with an average accuracy of \todo{XX\%}.

Refactorings

The models are able to predict three different refactorings levels: class-, method-, and variable-level. The following table presents such refactorings.

The 20 refactoring operations that our models are able to predict.
# Refactoring Problem Solution
Class-level refactorings
1 Extract Class A class is performing the work of two or more classes. Create a class and place the fields and methods relevant in it.
2 Extract Subclass A class owns features that are used only in certain scenarios. Create a subclass and use it in these scenarios.
3 Extract Super-class Two classes own common fields and methods. Create a super class and move the identical fields and methods.
4 Extract Interface A set of clients are using the same part of a class interface. Move the part portion to its own interface.
5 Move Class A class in a package among other classes not related. Move the class to a more relevant package.
6 Rename Class The class's name does not explain what the class does. Rename the class.
7 Move and Rename Class A class and its name among other classes are not related. Rename and move the class to a relevant package.
Method-level refactorings
8 Extract And Move Method A code fragment is scattering or tangling. Move the code to a new method and replace the old code's call.
9 Extract Method You have statements that can be grouped together. Move these statements to a separate new method.
10 Inline Method When the method's statements is obvious. Replace any calls to the method with the method's content.
11 Move Method A method is used more in the class than in its own class. Create a new method then move code.
12 Pull Up Method Sub-classes have methods that perform similar work. Turn the methods identical and move them to the super-class.
13 Push Down Method The behavior of a super-class is used in one or a few sub-classes. Move this behavior to the sub-classes.
14 Rename Method The name of a method does not explain the method's purpose. Rename the method.
Variable-level refactorings
15 Extract Variable There is an expression that is hard to understand. Put the outcome of the expression in separate variables.
16 Inline Variable A variable is used to assign a result. Replace the references to the variable with the expression itself.
17 Parameterize Variable bla bla bla bla bla bla
18 Rename Parameter The name of a parameter does not explain its purpose. Rename the parameter.
19 Rename Variable The name of a variable does not explain the variable's purpose. Rename the variable.
20 Replace Variable with Attribute bla bla bla bla bla bla

How to use

Restfull API

We created an API to make easier to use our models to predict refactorings. Basically the user just need to call an URL and pass some informations.

URL to call:

GET ---> http://0.0.0.0:5000/api/v1.0/recommend

Get information about how to use the models.

Output JSON
 {
  "About:": "This is an API that explores the effectiveness of machine learning models to predict software refactorings. More specifically, we make use of six different machine learning algorithms (i.e., Logistic Regression, Naive Bayes, SVM, Decision Trees, Random Forest, and Deep Neural Network) and train them in a dataset of more than two million real-world refactorings that happened in 11,149 projects  from the Apache, F-Droid, and GitHub ecosystems. ",
  "Authors": "Anonymous authors",
  "How to use": "For more information how to use this api, please visit www.refactoring.io."

POST ---> http://0.0.0.0:5000/api/v1.0/recommend
